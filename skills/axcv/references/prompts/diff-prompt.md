# Diff Prompt 模板

生成仅针对 [具体行为/功能] 的最小补丁。

## 必须遵守的约束

在生成补丁前，阅读 `references/project-constraints.md` 获取最新约束。

核心约束摘要：
- 框架：Astro（SSR），组件用 `.astro` 格式
- DB：CloudBase 云函数 / NoSQL，禁止 `prisma db push`
- UI：移动端优先，禁用下拉框（改用按钮点选）
- 资源：图片用 `.webp` 格式
- 文件行数限制：代码/文档均不超过 500 行
- 文档先行：开发前必读 `docs\文档索引.md`
- 变更同步：增删改功能时同步更新 `docs` 目录

## 要求

1. 不要重写周围代码
2. 仅输出统一 Diff 格式的补丁
3. 每个变更必须有明确的理由（引用规范文件中的约束或验收检查）

## 占位符

- 目标文件：[文件路径]
- 变更行为：[具体描述]
