# Test Prompt 模板

为此功能生成测试。

## 必须遵守的约束

在生成测试前，阅读 `references/project-constraints.md` 获取最新约束。

核心约束摘要：
- E2E 测试用 Playwright，必须模拟真实浏览器操作
- Astro SSR 页面需验证服务端渲染输出
- CloudBase DB 操作需验证边界条件（空值/权限/并发）

## 要求

1. 至少包含一个针对 [已知边界情况/已知陷阱] 的失败用例
2. 测试必须覆盖规范文件中定义的所有验收检查
3. 使用项目现有的测试框架和约定

## 占位符

- 目标文件：[文件路径]
- 测试重点：[具体描述]
- 已知边界情况：[从规范文件的"已知陷阱"中提取]
